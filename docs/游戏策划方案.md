## 一、游戏概述

### 1.1 游戏名称
**Letter-Blast** - 字母快打

> **说明**: 本文档是游戏的完整策划方案，后续将根据此方案进行实现。当前项目已有基础框架，重点是完善 GameMechanics 和添加新功能。

### 1.2 游戏类型
单词解谜游戏 (Word Search Puzzle)

### 1.3 核心玩法
在4×18的字母网格中，通过连接相邻格子（上下左右）拼写出单词。找到的单词会使格子变灰且不可再用。游戏以故事 progression 的方式展开，每关需要找出3-10个特定单词。玩家在有限时间内尽可能获得高分。

### 1.4 美术风格
采用**现代怀旧 2.5D 像素风**，结合3D空间中的2D像素精灵与现代物理光照，营造青春校园怀旧氛围。详见：`docs/美术风格.md`

### 1.5 项目现状
项目已有基础框架（scripts/, scenes/, data/），实现了：
- ✅ 基础网格生成
- ✅ 单词放置算法
- ✅ 输入处理与路径验证
- ✅ 单词验证系统
- ✅ UI显示

待实现功能：

- ⏳ 得分系统优化

## 二、核心机制

### 2.1 网格系统

| 属性 | 值 |
|------|-----|
| 网格尺寸 | 4列 × 18行 |
| 格子尺寸 | 60×60 像素 |
| 连接方向 | 上、下、左、右（十字形，无斜向） |
| 最小单词长度 | 2个字母 |
| 最大单词长度 | 8个字母 |

### 2.2 单词选择系统

**单词库**：当前包含21个水果类单词
```
APPLE, BANANA, CHERRY, DATE, ELDERBERRY, FIG, GRAPES, HONEYDEW, 
KIWI, LEMON, MANGO, NECTARINE, PAPAYA, QUINCE, RAISIN, 
STRAWBERRY, TANGERINE, ULUBAR, VANILLA
```

**关卡难度分级**：
- **Level 1-3**: 3个单词（平均长度 4-5 字母）
- **Level 4-6**: 4-5个单词（平均长度 5-6 字母）
- **Level 7-10**: 5-6个单词（混合长度 4-7 字母）
- **Level 11+**: 6-10个单词（混合长度 5-8 字母）

**故事 progression**：玩家在游戏过程中逐步解锁单词库中的单词，每个关卡代表一个特定的单词组合，形成“收集单词”的叙事线。

### 2.3 时间与得分系统

**单关时长**：3-5分钟（平衡模式）

**得分规则**（加权计分）：
| 单词长度 | 基础分 | 难度系数 | 实际得分 |
|----------|--------|----------|----------|
| 2-3字母 | 10分 | ×1.0 | 10分 |
| 4-5字母 | 20分 | ×1.5 | 30分 |
| 6-7字母 | 40分 | ×2.0 | 80分 |
| 8+字母 | 80分 | ×3.0 | 240分 |

**时间奖励**：
- 每剩余1秒 = 0.5分（四舍五入）
- 完美通关（剩余时间 > 60秒）= 额外100分

### 2.4 游戏流程

```
游戏开始 → 选择关卡 → 生成4×18网格 → 3-10分钟解谜 → 找到所有单词 → 
进度解锁 → 显示得分 → 进入下一关 → (循环) → 游戏结束/最高关卡
```

### 4.3 关键算法

#### 单词放置算法
```
for 每个需要放置的单词:
    重复 MAX_PLACEMENT_ATTEMPTS 次:
        1. 随机选择起始坐标 (x, y)
        2. 随机选择方向 (0:下, 1:右, 2:上, 3:左)
        3. 检查是否可以放置（边界检查 + 冲突检查）
        4. 如果可以放置，则执行放置
    如果100次尝试后仍无法放置，调整网格或跳过
```

#### 路径验证算法
```
function is_valid_path(path):
    if path.length < 2: return false
    
    for i from 1 to path.length - 1:
        current = path[i]
        previous = path[i-1]
        
        if current in path[0:i]: return false  # 无自相交
        
        dx = abs(current.x - previous.x)
        dy = abs(current.y - previous.y)
        
        if dx > 1 or dy > 1: return false  # 必须相邻
        if dx == 0 and dy == 0: return false  # 不能原地
    
    return true
```

#### 得分计算算法
```
function calculate_score(word):
    length = word.length()
    
    if length <= 3:
        return 10 * 1.0
    elif length <= 5:
        return 20 * 1.5
    elif length <= 7:
        return 40 * 2.0
    else:
        return 80 * 3.0
```

## 五、关卡设计

### 5.1 初始关卡示例

**Level 1 - Unit 1: Basic Fruits**
- 目标：熟悉游戏机制
- 难度：简单
- 单词数：3个
- 单词组合：`[APPLE, Kiwi, LEMON]`
- 推荐时间：120秒

**Level 2 - Unit 2: Common Words**
- 难度：简单-中等
- 单词数：4个
- 单词组合：`[BANANA, DATE, FIG, GRAPES]`
- 推荐时间：150秒

**Level 3 - Unit 3: medium Challenge**
- 难度：中等
- 单词数：5个
- 单词组合：`[CHERRY, HONEYDEW, MANGO, NECTARINE, PAPAYA]`
- 推荐时间：180秒

## 六、扩展功能（Future Plan）

### 6.1 难度调节机制
- **简单模式**：单词长度3-5，方向仅水平/垂直
- **普通模式**：单词长度3-6，允许斜向
- **困难模式**：单词长度5-8，所有8个方向，单词可能重叠

### 6.2 提示系统
- 每关3次提示机会
- 提示类型：高亮一个单词、揭示一个字母、移除一个错误单词

### 6.3 键盘支持
- 键盘方向键导航
- 回车键确认
- Esc键取消选择

### 6.4 成就系统
- 词语收集者：找到所有水果单词
- 连击大师：连续10次不犯错
- 时间大师：在50%时间内完成所有关卡

### 6.5 多语言支持
- 英文（当前）
- 中文（翻译）
- 日文（翻译）
