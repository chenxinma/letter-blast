## 一、主游戏场景增强：提示模式切换

### 1.1 功能概述

在主游戏界面增加提示模式开关，允许玩家选择单词提示的显示方式，从"拼写记忆"扩展到"含义记忆"。

### 1.2 提示模式定义

| 模式 | 显示内容 | 学习目标 | 难度 |
|------|----------|----------|------|
| **中英文模式**（默认） | 显示英文单词 + 中文释义 | 拼写 + 含义对照 | 简单 |
| **仅中文模式** | 仅显示中文释义 | 含义→拼写回忆 | 中等 |
| **仅英文模式** | 仅显示英文单词 | 原有玩法（拼写辨认） | 简单 |

### 1.3 界面设计

```
┌─────────────────────────────────────────┐
│ 分数：                                  │
│   ┌───┬───┬───┬───┬───┬───┬───┬───┐     │
│   │ A │ P │ P │ L │ E │ B │ K │ Q │     │
│   ├───┼───┼───┼───┼───┼───┼───┼───┤     │
│   │ ...                                  │
│   └───┴───┴───┴───┴───┴───┴───┴───┘     │
│                                         │
│   目标单词: [ 苹果 ]          ← 仅中文模式│
│   或者                                 │
│   目标单词: [ APPLE (苹果) ]  ← 中英文模式│
│                                         │
├─────────────────────────────────────────┤
│  提示模式: [中英文] [仅中文] [仅英文]     │
│  (切换按钮，当前选中高亮)                 │
└─────────────────────────────────────────┘
```

### 1.4 交互逻辑

- **默认模式**：中英文模式（保持现有体验，降低入门门槛）
- **模式切换**：点击即时生效，当前关卡继续进行
- **状态记忆**：用户选择保存至本地配置，下次启动沿用
- **仅中文模式判定**：玩家在网格中找到正确单词后，与 words.json 中的 en 字段匹配

### 1.5 数据依赖

完全复用现有 `data/words.json` 格式：

```json
{
  "words": [
    {"en": "APPLE", "zh": "苹果"},
    {"en": "BANANA", "zh": "香蕉"}
  ]
}
```

无需额外数据字段，零用户负担。

### 1.6 技术实现要点

- 新增 `ui_manager.gd` 中的提示模式状态管理
- 新增 `SettingsManager` 类管理用户偏好设置
- UI 增加模式切换按钮组件
- 目标单词显示逻辑根据模式动态渲染

## 二、新增场景：连线游戏（记忆强化）

### 2.1 功能概述

作为一局游戏的**总结场景**，在玩家完成网格搜索后进入。通过"单词-释义连线"强化记忆效果，形成完整的学习闭环。

### 2.2 场景流程

```
主游戏（网格搜索）
       ↓
   找完所有单词
       ↓
┌────────────────────────┐
│   连线游戏（总结场景）    │
│   - 复习本局遇到的单词    │
│   - 单词与释义配对       │
│   - 强化记忆效果        │
└────────────────────────┘
       ↓
   显示得分 & 学习进度
       ↓
   进入下一局 / 返回主菜单
```

### 2.3 连线游戏设计

#### 玩法说明

- 显示本局游戏遇到的 4 个单词（左侧）及其释义（右侧，打乱顺序）
- 玩家点击左侧单词，再点击右侧释义进行连线
- 连线正确显示绿色 ✓，错误显示红色 ✗
- 全部连对后进入得分结算界面

#### 界面布局

```
┌─────────────────────────────────────────┐
│         🎯 记忆强化 - 连线挑战           │
├─────────────────────────────────────────┤
│                                         │
│   ┌─────────┐          ┌─────────┐     │
│   │ APPLE   │────────── │ 香蕉    │     │
│   └─────────┘          └─────────┘     │
│                                         │
│   ┌─────────┐          ┌─────────┐     │
│   │ BANANA  │          │ 苹果    │     │
│   └─────────┘          └─────────┘     │
│                                         │
│   ┌─────────┐          ┌─────────┐     │
│   │ CHERRY  │          │ 樱桃    │     │
│   └─────────┘          └─────────┘     │
│                                         │
│   ┌─────────┐          ┌─────────┐     │
│   │ MANGO   │          │ 芒果    │     │
│   └─────────┘          └─────────┘     │
│                                         │
├─────────────────────────────────────────┤
│         进度: 2/4    正确率: 100%        │
└─────────────────────────────────────────┘
```

#### 计分规则

| 指标 | 分值 |
|------|------|
| 连线正确 | +20 分/词 |
| 连线错误（重试） | -5 分 |
| 全对奖励（无错误） | +50 分 |

### 2.4 记忆强化效果

- **即时复习**：刚在网格中找过的单词，立刻在连线中再次出现
- **双向联想**：看单词想释义，强化含义记忆
- **主动回忆**：比被动识别更有效的记忆方式
- **成就感**：完成连线后获得额外分数，增加正向反馈

### 2.5 技术实现要点

- 新增场景 `scenes/matching_game.tscn`
- 新增脚本 `scripts/matching_game.gd`
- 连线数据来自本局 `leitner_manager` 中使用的单词
- 连线动画使用 Godot 的 Line2D 或 Tween 实现
- 跨场景数据传递：通过全局单例或信号机制

## 七、总结

本补充稿在现有 Letter-Blast 基础上，提出了三个核心增强方向：

1. **提示模式切换**：零数据成本，实现"含义→拼写"的学习模式
2. **连线游戏**：作为总结场景，形成"找词→连线"的学习闭环
3. **学习进度界面**：可视化展示，增强用户成就感和持续学习动力

三个功能模块均复用现有 `words.json` 数据格式，不增加用户准备内容的负担，符合项目"易于扩展"的设计原则。
